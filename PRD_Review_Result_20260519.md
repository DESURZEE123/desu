# 📋 美食收藏小程序 PRD V1.0 评审结论

## 总体判定

**结论：❌ 不通过 — 需修复阻断项后复评**

> **核心理由**：MVP 功能范围出现严重漂移，两个核心功能（地图模式、附近推荐）从 P0 降为搁浅状态，但文档未相应调整；同时新增的搜索功能、距离展示、定位逻辑调整未明确定义，冲击了需求清晰度和可开发性。

---

## 🔴 阻断项（5 条）

### 1. F6/F11 功能状态混淆
- **问题**：地图模式和附近推荐标记为 P0 但已搁浅，与"地图化"产品定位冲突
- **修改建议**：决策 (A) 从 V1.0 MVP 移除，更新 2.6、3.6、验收用例；(B) 或保留时间表说明何时落地

### 2. 距离展示需求不明确
- **问题**：用户澄清列表需展示距离，但文档未定义列表距离的位置、计算方式、精度
- **修改建议**：补充列表卡片的距离字段设计、精度定义（精确到几米）、计算方式文档

### 3. 搜索功能拆分为两入口但定义缺失
- **问题**：列表下方"搜索特定数据"与右上方"全局搜索"的范围、交互、API 调用完全不清楚
- **修改建议**：补充两个搜索框的完整定义及交互关系，明确 API 设计

### 4. 地图 Tab 处理无决策
- **问题**：地图功能搁浅但首页框架仍占位"地图/列表切换"，UI 状态不明确
- **修改建议**：决策隐藏地图 Tab 还是保留为禁用状态，更新首页框架描述

### 5. 定位授权逻辑调整未同步文档
- **问题**：用户澄清改为首次一次性获取，但 PRD 流程图、前置条件、异常处理仍描述旧逻辑
- **修改建议**：更新 2.2.1 流程图、3.1.3 前置条件、2.3.1 异常处理

---

## 🟡 重要项（6 条）

### 1. 搁浅功能的数据库字段处理不明确
地图/附近推荐搁浅，但 `longitude`、`latitude` 字段仍在数据字典。建议保留字段以支持后续版本。

### 2. 验收用例与实际功能不符
用例 7 "附近推荐" 涉及搁浅功能。如 F11 从 V1.0 移除，需删除此用例。

### 3. 列表无坐标记录的展示方式不明确
快速文字记录可能无坐标，列表/地图的展示规则不完整。需补充设计稿说明。

### 4. 搁浅功能的埋点处理缺失
如移除附近推荐，应删除 `nearby_view`、`nearby_click` 埋点定义。

### 5. 隐私合规风险
定位数据收集即使一次性，仍需隐私政策覆盖采集时机、用途、存储期限等。

### 6. MVP 性能目标中地图相关指标
如地图搁浅，应删除"地图加载 ≤1.5s"指标。

---

## 🟢 建议项（3 条）

1. 搜索框在列表中的 UI 位置需明确可视化（wireframe 中标注）
2. 距离计算算法需补充（直线 vs 步行距离、刷新策略）
3. 灰度策略虽是个人项目，但应有基础回滚方案

---

## 各角色简评

| 角色 | 判定 | 核心问题 |
|------|------|---------|
| 产品 | ❌ | MVP 范围混乱；附近推荐和地图模式搁浅与产品定位冲突；新增搜索需求定义不完整 |
| 研发 | ❌ | 定位逻辑未同步文档；距离计算需求不明确；两个搜索 API 设计缺失；字段处理不确定 |
| 测试 | ❌ | 验收用例 7 失效；距离验收标准缺失；搜索测试用例完全缺失；地图 Tab 验收标准无 |
| 设计 | ❌ | 地图 Tab 处理不确定；列表搜索框位置不明确；距离展示细节缺失；两个搜索结果样式可能冲突 |
| 运营 | ❌ | 埋点处理缺失；距离埋点未定义；搜索埋点需补充；灰度策略仅说后续；数据指标口径不清 |
| 法务 | ❌ | 定位隐私风险不能忽略；图片鉴黄推到 V1.1 有风险；用户注销数据清理规则需完善 |
